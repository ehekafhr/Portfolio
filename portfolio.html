군 경계지역 CCTV 영상 품질 개선을 통한 지능형 분석 고도화

부스트캠프 AI Tech 8기 최종 프로젝트 5인 팀 GitHub ↗ 시연 영상 ↗

군 경계 지역 CCTV의 화질 열화(해무·안개 등)를 엣지 디바이스에서 실시간 복원하고, 객체 탐지 → HyperCLOVA X 기반 보고서 생성까지 이어지는 파이프라인을 구축해 군 경계 감시 업무의 자동화를 지향한 프로젝트입니다. 엣지 디바이스(Jetson Orin Nano / Google Coral) — 메인 서버 — 웹 서버로 구성된 End-to-End 시스템을 완성했습니다.

주요 기여

  • 메인 서버 구축(FastAPI) — 엣지 디바이스의 탐지 결과와 이미지를 수신해 AWS S3에 저장하고, LLM 보고서 생성과 웹 서버 조회 요청을 중계하는 메인 서버를 구현했습니다. 엣지 디바이스의 Heartbeat를 수신해 기기 연결 상태를 추적하는 로직도 포함했습니다.
  • DB 스키마 설계 및 구현(MongoDB) — 촬영 시각·위치·S3 경로·탐지 메타데이터·LLM 캡션·구조화 보고서를 담는 탐지 결과 스키마와, 엣지 디바이스 상태 스키마를 설계해 과거 탐지 기록 열람 기능의 기반을 만들었습니다.
  • HyperCLOVA X 파이프라인 연동 — 탐지 이미지를 HCX-005로 분석해 상황 캡션을 생성하고, 이를 HCX-007로 구조화된 JSON 보고서로 변환하는 2단계 LLM 파이프라인을 메인 서버·DB와 연동했습니다.
  • 데이터 수집 및 EDA — AI-Hub 군 경계 작전 환경 데이터(약 47,000장) 등 6종 데이터셋을 수집·분석하고, 클래스 분포를 유지하는 Group-Stratified Split(8:2)으로 학습/평가 데이터를 구성했습니다.

재활용 품목 분류를 위한 Object Detection

부스트캠프 AI Tech 8기 5인 팀 · 팀장 GitHub ↗

재활용 쓰레기가 포함된 이미지에서 객체별 Bounding Box와 품목 Label을 예측하는 대회 프로젝트입니다. 평가지표는 mAP50이며, 외부 데이터 없이 COCO 사전학습 가중치만 사용할 수 있는 제약이 있었습니다.

담당 역할 — 프로젝트 총괄 기획 · Co-DeTR 모델 학습 · WBF 앙상블

주요 기여

  • 모델 선정(Co-DeTR, Swin-L Backbone) — 팀원들이 다양한 모델을 학습·비교·앙상블해야 하는 상황에서, 코드 응집도와 재사용성을 위해 MMDetection을 팀 공통 기반으로 채택했습니다. Co-DeTR은 무겁지만 Gradient Accumulation으로 GPU RAM과 대회 기간을 모두 만족할 수 있다고 판단했고, 특히 BatchNorm 대신 LayerNorm을 사용해 작은 Batch에서도 Accumulation으로 큰 Batch와 유사한 효과를 낼 수 있다는 점을 근거로 선정했습니다.
  • 과적합 검증 기반 파인튜닝 전략 — COCO와 Label이 다른 Decoder는 전체 재학습하되, Backbone 전체 재학습은 과적합을 유발한다는 가설을 실험으로 검증했습니다. 학습 데이터에서는 전체 학습이 mAP50 +0.05 높았지만 검증 데이터에서는 마지막 Layer만 학습한 모델이 +0.03 높아, Backbone 마지막 Layer만 학습하는 전략을 채택했습니다.
  • WBF(Weighted Box Fusion) 앙상블 — 박스를 제거하는 NMS와 달리 신뢰도·위치·크기를 종합해 새 박스를 생성하는 WBF가 더 안정적일 것으로 가설을 세웠고, 검증 데이터에서 mAP50 +0.0465를 확인해 최종 채택했습니다. EDA 결과를 바탕으로 모델·클래스별 가중치를 부여하고, IoU Threshold는 Grid Search로 최적화했습니다.
  • 팀 리딩 — 약 8일의 짧은 대회 기간 내 완주를 위해 업무별 시한을 배정하고, 병목 방지를 위해 초기 모델 세팅에 인원을 집중 배치했습니다. Wandb로 실험을 관리하고 Notion에 결과를 정리해 앙상블 의사결정의 근거로 활용했습니다.

Hand Bone Image Segmentation

부스트캠프 AI Tech 8기 5인 팀 GitHub ↗

손 X-ray 영상(2048×2048)에서 29개 손뼈 클래스를 각각 Segmentation하는 프로젝트입니다. 뼈가 겹치는 영역은 여러 클래스로 동시에 라벨링해야 하는 multi-label 문제이며, 평가지표는 클래스 평균 Dice(mDice)입니다. (Train 800장 / Test 288장)

0.9759
최종 앙상블 Dice
29 classes
SAM-2 Decoder 다중 마스크 개조
담당 역할 — 모델 실험(SAM-2) · 앙상블 설계

주요 기여

  • SAM-2 선정 및 구조 수정 — 이미지 외에 프롬프트를 입력받아 추가 정보를 주입할 수 있다는 점에서 SAM-2를 선정했고, 픽셀당 하나의 클래스만 출력하는 Decoder를 29개 클래스별 마스크를 동시 출력하도록 개조했습니다.
  • 프롬프트 생성기 도입으로 학습 안정화 — 메타데이터의 직접 입력이나 뼈 이름 텍스트 프롬프트(finger-1, finger-2 등)는 인코더가 구분하기 어렵다고 판단했습니다. 프롬프트 없이 학습하면 초기 학습이 불안정해지는 문제를, 각 뼈의 중심점 좌표를 출력하는 프롬프트 생성기를 별도 학습해 프롬프트 인코더에 연결하는 방식으로 해결했습니다.
  • 고해상도 입력 처리 — 모델 크기로 인해 원본 해상도를 직접 학습할 수 없어 축소 입력 후 원본 크기로 복원하는 파이프라인을 구성했고, 크기 조정에는 Bilinear 대비 고품질인 Bicubic 보간을 사용했습니다. 추가 연산량은 무거운 모델 학습 대비 미미해 전체 학습 속도에 영향이 없다고 판단했습니다.
  • Dice loss 역수 기반 Weighted Ensemble — validation Dice를 그대로 가중치로 쓰면 모델 간 점수 차가 작아 단순 평균과 다를 바 없었고, 지수항만으로는 계산이 불안정해지는 10 이상의 지수가 필요했습니다. 이를 (1/(1−Dice))power 형태의 Dice loss 역수 가중치로 해결해 모델·클래스별 성능 차이의 영향력을 키웠고, power 5~10 구간에서 최고 성능 Dice 0.9759를 달성했습니다.

Gradual Restriction with SC2

DGIST 학부 강화학습 팀 프로젝트 GitHub ↗

강화학습 에이전트를 다른 환경으로 전환할 때, 환경 변화를 점진적으로 적용하며 파인튜닝하는 방식의 효과를 검증한 프로젝트입니다. 스타크래프트 2(Tencent TStarBot1 기반 macro-action 환경, PPO)를 기본 환경으로, APM(분당 액션 수)이 일정 값을 넘으면 제약을 부여하는 환경을 전환 대상으로 설정했습니다.

+25% / +10%
점진적 전환 시 승률 향상 (하수 / 고수 상대)
TF 2.8
레거시 코드 마이그레이션
담당 역할 — APM 제약 환경 설계 · 학습 실험 설계 · 코드 현대화

주요 기여

  • APM 제약 환경 설계 — 전략이 아닌 비인간적 실행 속도로 이기지 않도록 행동 횟수를 제한하는 reward function을 수정했습니다. 높은 APM에 페널티만 부여하면 중요한 순간에는 여전히 극단적 APM을 사용하는 문제가 있어, 순간적으로 높은 APM 사용 시 즉시 Action이 불가능해지는 강한 제약을 적용했습니다.
  • 점진적 환경 전환 실험 — 변경 환경을 즉시 전부 적용하는 파인튜닝과 점진적으로 변화시키는 학습을 비교했습니다. 급격한 변화는 기존 정책이 적응하지 못하고 점진적 변화는 단계적 적응을 가능하게 한다는 가설을 세웠고, 실험 결과 점진적 방식이 승률 +25%(낮은 실력 상대) / +10%(높은 실력 상대) 향상을 보여 가설을 검증했습니다.
  • 레거시 코드 현대화 — 구버전 TensorFlow 기반 TStarBot1 코드를 TensorFlow 2.8에서 동작하도록 마이그레이션했습니다.
